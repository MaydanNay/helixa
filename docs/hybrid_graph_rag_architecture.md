# Архитектура Гибридного Мозга (Hybrid GraphRAG) для Helixa

Этот документ описывает поэтапный план внедрения гибридной памяти (Граф Знаний + Векторная база) в микросервис `Helixa` с целью создания ИИ-агентов с человекоподобным уровнем логики, долговременной памяти и пространственно-временного осознания.

## 1. Концепция: Зачем нам Гибридный Мозг?

Чистая векторная память (Qdrant), которая используется сейчас, отлично справляется с семантическим поиском (нахождение похожих текстов) и извлечением эмоциональных "эпизодов". Однако она не способна к **многошаговому логическому выводу (Multi-hop reasoning)**. Вектор не понимает строгих связей (кто кому родственник, кто где живет, кто чем владеет) без потери контекста.

**Решение — Hybrid GraphRAG:**
*   **Neo4j (Граф Знаний — Семантическая память):** Хранит жесткие факты как узлы (Агент, Место, Предмет) и ребра (ДРУЖИТ_С, ВЛАДЕЕТ, ЖИВЕТ_В). Это позволяет агенту делать точные логические выводы (например, понимать социальные связи через "рукопожатия").
*   **Qdrant (Векторная БД — Эпизодическая память):** Хранит неструктурированные куски воспоминаний, диалогов, эмоций и размышлений в виде эмбеддингов.

При формировании ответа агент запрашивает **И Граф, И Вектор**, получая идеальную картину: непоколебимые факты + эмоциональный окрас из прошлого.

## 2. Архитектура и Инфраструктура

### 2.1 Обновление `docker-compose.yml`
Необходимо добавить контейнер с базой данных Graph DB (Neo4j) в общий стек `Helixa`.

```yaml
  neo4j:
    image: neo4j:5.23.0
    restart: always
    environment:
      NEO4J_AUTH: neo4j/${NEO4J_PASSWORD} # Задать пароль в .env
      NEO4J_dbms_memory_heap_max__size: 2G
    ports:
      - "7474:7474" # HTTP UI (Neo4j Browser - для визуализации графа)
      - "7687:7687" # Bolt port (для Python драйвера)
    volumes:
      - neo4j_data:/data
    networks:
      - helixa_default
```

*Не забыть добавить `neo4j_data` в секцию `volumes` в конце файла.*

### 2.2 Модели данных в Neo4j (Схема Графа)
Основные **Узлы (Nodes)**:
*   `Agent` (id, name, role)
*   `Location` (id, name, description)
*   `Item` (id, name, type)
*   `Event` (id, type, timestamp)

Основные **Связи (Edges/Relationships)**:
*   `(Agent)-[:KNOWS {trust: 0.8, relationship: "friend"}]->(Agent)`
*   `(Agent)-[:LIVES_IN]->(Location)`
*   `(Agent)-[:OWNS]->(Item)`
*   `(Agent)-[:PARTICIPATED_IN]->(Event)`

## 3. Поэтапный План Внедрения

### Этап 1: Инфраструктура и Драйверы (✅ ВЫПОЛНЕНО)
1.  [x] Добавить **Neo4j** в `docker-compose.yml` и обновить `README.md` по запуску.
2.  [x] Добавить библиотеку `neo4j` в зависимости проекта (в `Dockerfile` или `requirements`).
3.  [x] Создать модуль `app/services/neo4j_client.py` (по аналогии с `qdrant_client.py`).
    *   Реализовать класс `GraphMemoryService` с методами инициализации соединения.
    *   Добавить базовые методы (CRUD для узлов и ребер): `create_agent_node`, `create_relationship`, `get_agent_subgraph`.

### Этап 2: Интеграция в пайплайн создания новых агентов (✅ ВЫПОЛНЕНО)
Процесс в `worker.py` (функции генерации) нужно синхронизировать с графом:
1.  [x] **Создание узла (Node):** При первичной генерации "Души" (Soul) сразу создавать узел `(:Agent)` в Neo4j со ссылкой на его внутренний UUID.
2.  [x] **Экстракция Биографии (Knowledge Extraction):** Создан сервис `kg_extractor.py`. После генерации жизненного цикла (Lifecycle/Background), сгенерированный текст передается в LLM с использованием Structured Outputs.
3.  [x] **Построение Графа Лора:** LLM возвращает JSON со списком сущностей (родственники, враги, города) и связей. Этот JSON парсится и превращается в узлы и связи вокруг нового агента в Neo4j (метод `ingest_knowledge_graph`).

### Этап 3: Pipeline Обновления Памяти Симуляции (Update Memory / Stream) (✅ ВЫПОЛНЕНО)
Здесь `Helixa` связывается с Mimora (симулятором):
1.  [x] **Эндопинт для Потока Жизни:** Создан API `/agents/{agent_id}/memory/stream`, куда симуляция шлет "События" (например: "Иван купил эликсир у Анны в 14:00").
2.  [x] **Эпизодическая запись (Qdrant):** Событие прогоняется через модель эмбеддингов и ложится в вектор как эпизод в коллекцию агента.
3.  [x] **Формирование фактов (Neo4j):** Это же событие подается в `kg_extractor`. LLM решает: нужно ли создать новую связь. Сценарий сохраняется в Neo4j через `MERGE`.

### Этап 4: Механизм Воспоминания (Hybrid Retrieval) при генерации действий (✅ ВЫПОЛНЕНО)
Когда архитектуре нужно понять, что агент сделает дальше, или как он ответит в чате:
1.  [x] **Шаг 1. Графовый Контекст (Cypher Query):**
    Запрос в Neo4j: "Найти самого агента и его подграф (соседи 1-го и 2-го уровня), с кем он знаком, где он сейчас". Реализован метод `retrieve_agent_context`.
2.  [x] **Шаг 2. Векторный Контекст (Semantic Search):**
    Запрос в Qdrant по текущему контексту (найти релевантные прошлые разговоры или эмоции об Анне).
3.  [x] **Шаг 3. LLM Action Generation:**
    Создан API `/agents/{agent_id}/action`. И граф (Факты), и вектор (Эмоции) сливаются в один массивный `System Prompt`. Агент генерирует действие, учитывая как логику мира, так и свои субъективные переживания.

## 4. Спецификация Knowledge Extractor (Извлечение Графа из Текста)

Ключевой брокер между свободным текстом и строгим Neo4j — это **Knowledge Extractor**, использующий Structured Output от LLM.
Нужно создать `app/services/kg_extractor.py`.

Пример системного промпта для модуля:
```text
Ты специалист по извлечению информации в виде Графа Знаний (Knowledge Graph).
Извлеки из текста все сущности и их отношения, чтобы наполнить базу данных симуляции мира.

Разрешенные типы узлов (nodes): AGENT, LOCATION, ITEM, ORGANIZATION, EVENT.
Разрешенные типы связей (edges): LIVES_IN, KNOWS, OWNS, WORKS_AT, VISITED, PARTICIPATED_IN, DISLIKES, LIKES.

Ответ должен быть строго в формате JSON:
{
  "nodes": [
    {"id": "Иван", "type": "AGENT"}, 
    {"id": "Пекарня", "type": "LOCATION"}
  ],
  "edges": [
    {"source": "Иван", "target": "Пекарня", "type": "VISITED", "properties": {"time": "вчера"}}
  ]
}
```

## 5. Преимущества для экосистемы (Mimora & Helixa)
1. **Глобальный Социальный Граф:** Прямой доступ к аналитике всей деревни. "Кто самый тайный лидер?" "Кто знаком со всеми?" — это мгновенные `Cypher` запросы, не требующие LLM-вычислений.
2. **Нулевые Галлюцинации по Фактам:** Агент никогда не придумает брата, которого нет в графе. Среда (Mimora) может жестко валидировать генерацию.
3. **Графовые Триггеры Симуляции:** Симуляция может автоматически реагировать на граф. Например, если два "врага" (связь `DISLIKES`) оказались в одной `LOCATION`, движок симуляции может принудительно запустить ивент "Случайная Стычка".

## 6. Резюме
Внедрение Hybrid GraphRAG делает `Helixa` полноценным **Символьно-Нейронным Движком (Neuro-Symbolic Engine)**.
Агенты обретают:
*   **Детерминированную логику и фактологию** (благодаря Neo4j).
*   **Человеческую интуицию, ассоциативность и эмоции** (благодаря Qdrant + LLM).
Это абсолютный state-of-the-art архитектурный паттерн для создания правдоподобной симуляции общества (Artificial Society).
